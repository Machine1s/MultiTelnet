# MultiTelnet 项目学习与进阶计划表

本计划旨在帮助你从“代码使用者”成长为“架构理解者”。通过五个阶段的拆解，你将掌握 Python 自动化运维工具的核心技术。

---

## 📅 第一阶段：数据模型与配置解析 (地基篇)
**目标**：掌握如何处理结构化配置信息。

- [ ] **1. YAML 深入理解**：阅读 `inventory/hosts.yaml`。
    - 知识点：层级嵌套、标量/列表/字典的区别。
- [ ] **2. 语法糖：@dataclass**：阅读 `core/inventory.py` 顶部的 `HostConfig`。
    - 知识点：理解它是如何自动生成 `__init__` 等模板代码的。
- [ ] **3. 业务逻辑：Credentials 覆盖**：研究 `InventoryManager._load` 方法。
    - 知识点：学习如何处理“组凭据”与“单主机独立凭据”的优先级覆盖逻辑。
- **🧪 实验**：在 `hosts.yaml` 中新增一个字段（如机房位置 `loc`），并尝试在代码中打印出来。

---

## 🎨 第二阶段：CLI 交互与终端美学 (界面篇)
**目标**：掌握构建专业命令行工具（CLI）的方法。

- [ ] **4. Click 库的装饰器魔法**：详细研读 `manager.py` 下的 `@click.group`、`@cli.command`。
    - 知识点：理解如何定义子命令、选项（Option）以及布尔开关（Flag）。
- [ ] **5. Rich 可视化布局**：查看 `manager.py` 中的 `Table` 和 `Panel` 使用。
    - 知识点：学习如何在黑乎乎的滚动屏上创建带有颜色、边框和列对齐的精美表格。
- [ ] **6. 异步 UI：Progress 进度条**：研究 `Progress` 与 `executor.run_batch` 的配合。
    - 知识点：学习如何在后台线程干活时，前台界面依然能实时滚动进度条。

---

## ⚙️ 第三阶段：并发执行引擎 (核心篇)
**目标**：掌握高效的远程控制与并发处理技术。

- [ ] **7. 线程池：ThreadPoolExecutor**：阅读 `core/executor.py` 中的 `run_batch`。
    - 知识点：理解并发（Concurrency）的原理，以及如何收集异步执行后的各个结果。
- [ ] **8. 网络驱动：Netmiko 实战**：研究 `ConnectHandler` 的参数配置。
    - 知识点：区分 SSH 和 Telnet 处理逻辑，理解 `device_type` 的重要性。
- [ ] **9. “强力模式”调优**：回顾现场 Telnet 报错后我们做的修改：`expect_string` 与 `global_delay_factor`。
    - 知识点：学习如何针对现实中延迟高、提示符不标准的极端环境进行代码调优。

---

## 🧠 第四阶段：数据分析与审计逻辑 (智能篇)
**目标**：把原始数据变成有用的决策信息。

- [ ] **10. 异构分析算法**：阅读 `ResultAnalyzer.group_by_output`。
    - 知识点：学习如何通过“哈希归类”在 40 个结果中瞬间揪出那个“害群之马”。
- [ ] **11. 文本提取技术：HealthParser**：精读 `core/analyzer.py` 内的解析逻辑。
    - 知识点：掌握字符串切割（`split`）、成员检查（`in`）等数据解析技巧。
- [ ] **12. 日志持久化**：研究 `SimpleLogger`。
    - 知识点：学习如何使用内置 `csv` 库生成审计报表，理解“追加”与“覆盖”的存储区别。

---

## 🚀 第五阶段：工程化、环境与分发 (落地篇)
**目标**：完成从“代码”到“产品”的跨越。

- [ ] **13. Docker 模拟集群**：阅读 `simulation/Dockerfile` 和 `gen_simulation.py`。
    - 知识点：学习如何用代码“凭空”造出 10 台各具特色的服务器用于测试。
- [ ] **14. Git 版本控制流**：回顾你操作过的 `add`, `commit`, `branch`, `merge`。
    - 知识点：理解“提交历史”比“当前代码”更重要的意义。
- [ ] **15. PyInstaller 一键打包**：研读 `build_exe.py`。
    - 知识点：掌握环境变量处理、静态资源拷贝以及二进制构建流程。

---

### 💡 建议
每完成一个阶段，都尝试自己动手给项目加一个小功能。这种**“边拆解边重组”**的方法，是学习编程最快的捷径。
